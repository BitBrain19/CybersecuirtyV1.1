================================================================================
STEP 1: WHERE DO I PUT THE DATA?
================================================================================

Think of it like this: You have a toy box (your computer)
Inside the toy box, there's a folder called "ml" (the machine learning folder)
Inside "ml", there's a folder called "artifacts" (where data lives)
Inside "artifacts", there are more folders like:
  - datasets/
  - models/
  - logs/

TO PUT YOUR DATA:
1. Go to: d:\Cybergardproject_V1.1\ml\artifacts\datasets\
2. Put your data files there (CSV, JSON files with security events)
3. The computer will find them automatically

WHAT KIND OF DATA?
The system needs security event logs like:
  - Who logged in? (username, time, location)
  - What files did they access? (which files, how much data)
  - What commands did they run? (on the endpoint)
  - Which servers did they connect to? (network traffic)

EXAMPLE DATA FORMAT (Easy one):
  User,Time,Location,Action,Success
  john_smith,10:30AM,NYC,login,yes
  john_smith,10:45AM,NYC,file_read,yes
  john_smith,11:00AM,NYC,file_write,yes
  hacker_joe,2:00AM,China,file_read,yes
  hacker_joe,2:05AM,China,password_change,yes

PROBLEMS & SOLUTIONS
================================================================================

PROBLEM: "ModuleNotFoundError: No module named 'tensorflow'"
SOLUTION: Run this command again:
          python -m pip install -r requirements.txt

PROBLEM: "Permission denied"
SOLUTION: Make sure you're in the right folder (use cd command)
          Or run PowerShell as Administrator

PROBLEM: "File not found"
SOLUTION: Your data files aren't in the right place
          Put them in: d:\Cybergardproject_V1.1\ml\artifacts\datasets\

PROBLEM: "Out of memory"
SOLUTION: Your dataset is too big
          Start with smaller dataset first
          Or restart your computer

PROBLEM: "Training is really slow"
SOLUTION: That's normal! Go get coffee â˜•
          Big datasets take time
